# lottoMaker v1
> 로또 번호 생성기

---

## 1. 요구사항

- 유효범위 검증 : 정해진 범위 안의 값만 사용한다. (1~45)
- 중복 불가 : 값은 모두 달라야 한다. 
- 정렬 출력 : 결과는 항상 정렬된 형태로 출력해야 한다.
- 책임 분리 : 값, 묶음, 생성 책임을 서로 다른 객체로 나눈다.
- 예외 처리 : 잘못된 값이 들어오면 예외로 처리한다.
- 테스트 검증 : 핵심 요구사항은 테스트로 검증한다. 

---

## 2. 객체 설계 
- 값, 묶음, 생성을 분리해서 설계

### 도메인 모델 및 역할
- NumberValue (값) : 값 하나를 표현, 생성 시점 시 유효성 검증, 불변성 유지
- NumberTicket (묶음) : 유효성 검증 및 예외처리, 정렬
- NumberGenerator (생성) : NumberTicket을 생성하는 팩토리 객체

### 🔗 객체 간의 관계 
[ Main ] 
──(사용)──>[ NumberGenerator ] ──>
──> [ NumberValue ] (1~45 객체 HashSet으로 6개 수집) ──>
──> (TreeSet으로 정렬 및 주입)[ NumberTicket ] (UnmodifiableSet으로 불변 반환) ──>
──(반환)──>[ Main ]

테스트 코드로 테스트.

## 4. 컬렉션 선택 이유
- HaseSet -> "무작위 숫자를 뽑을 때 값이 모두 달라야 한다." 중복 제거에 가장 효율적이라고 생각해서 선택 
- TreeSet -> "결과는 항상 정렬된 형태" 내부 자료구조로 데이터를 저장할 때 자동적으로 정렬을 유지해줘서 선택, 별도의 정렬 로직 없이 처리

## 5. 고찰
- 같은 값이면 같은 객체로 판단해야 한다는 요구사항, 동등성 판단 equals, hashCode를 오버라이딩 했어야 했는데, 
- 고정적인 로또 번호라는점을 생각하고 매번 실행 할 때 마다 new로 객체를 만드는 방식보다는 클래스 생성 시점에
- 메모리에 올려두고 뽑아오는 방식이 좋다고 생각했다, 그럼 동일한지 확인하는 과정을 거치지 않아도 되니까

## 6. 어려웠던 점
