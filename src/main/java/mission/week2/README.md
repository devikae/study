# lottoMaker v2
> 로또 맞추기(수동)

---

## 1. 요구사항

- 입력된 값 프로그램에서 처리.
- 허용되지 않은 입력은 예외로 처리. 
- 같은 값이 반복되는 경우를 검증.
- 두 값 묶음을 비교 할 수 있어야 한다.
- 비교 결과를 별도 객체로 표현. 

---

## 2. 객체 설계
- 입력, 입력 값 묶음, 비교로 분리

### 도메인 모델 및 역할
- InputParser : 입력을 받고, 입력 값의 파싱 및 검증 처리
- TicketWallet : inputParser로 넘겨 받은 입력된 값들을 묶음 처리
- MatchResult : 입력 받은 값을 당첨과 비교 

## 3. v1과 달라진점
- 자동 생성이 아닌 사용자의 입력값을 직접 받고, 당첨 번호와 비교 결과 리턴

## 4. 입력 처리 방식
- BufferedReader, StringTokenizer 사용

## 5. 검증 기준
- 입력된 값이 있는지
- 1~45의 범위로 부터 벗어난 값은 아닌지
- 입력된 값에 중복이 있는지 

## 6. 예외 처리방식
- UncheckedException 처리
- 입력된 값의 null 체크
- 입력된 값의 범위 체크 
- 입력된 값이 중복이라면 사용자에게 알림

## 7. 비교 로직
1. matchResult 생성 시 당첨 번호 저장
2. 당첨 번호 저장 후 당첨 번호에서 보너스 번호 랜덤 선별
3. 입력된 번호와 당첨 번호 retainAll 로 교집합만 남김
4. 그 교집합의 수로 등수 판별 
5. 2,3등의 경우 보너스 번호의 포함 유무로 판별

## 8. 테스트 케이스 목록
1. InputParser
    - null 입력, 0, -1, 중복된 숫자, 가장큰 양수, 음수
2. MatchResult
   - 맞춘 번호의 비례한 등수

## 9. 어려웠던 점
문자열을 어떻게 받을지, 구분은 어떻게 할껀지, 객체가 작게 나뉘고 묶는 객체를 따로 두니까
컬렉션을 쓸 때 맞춰줘야하는데 내가 만들었음에도 자꾸 헷갈린다, 파싱은 어떻게 할지 
데이터를 넘겨줄 때 어떻게 넘겨줄지 기존 구조는 헤치지 않고 어떻게 만들지 고민을 많이 했다
